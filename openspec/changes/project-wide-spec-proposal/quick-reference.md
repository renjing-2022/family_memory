# 项目规范速查表（第一版）

> 适用范围：`project-wide-spec-proposal` 变更实施全流程。  
> 使用方式：开发、联调、评审、验收时对照此页快速自检。

## 1) 设计稿与视觉规范

- 设计稿固定目录：`design/`
- 版本命名规则：`YYYY-MM-DD-vN`（示例：`2026-04-22-v1`）
- 必做映射：每个前端页面必须建立“设计稿页面 -> 代码页面”映射表
- 评审口径：视觉表现遵循设计稿，交互与业务行为遵循 OpenSpec `specs`

## 2) 主题规范（含 token）

- 主题模式范围：浅色、深色、系统跟随
- 持久化策略：账号维度跨端同步（同账号在不同终端保持一致主题）
- 主题令牌来源：`design/token.css`
- 禁止项：组件内硬编码主题颜色值（例如直接写十六进制颜色替代语义变量）
- 建议项：优先使用语义令牌（如文本、背景、边框）而非临时色值

## 3) 上传与媒体治理

- 上传默认阈值（当前冻结）：
  - 格式：`jpg`、`png`、`webp`
  - 大小：单文件 `10MB`
- 失败策略：上传失败不阻塞“标题 + 日期”最简保存，支持后续补传
- 生命周期治理：记录删除或失效后，关联媒体按清理策略标记并周期清理

## 4) 性能与可访问性基线

- 首屏可交互：不超过 `3 秒`
- 关键操作反馈：不超过 `1 秒` 显示处理状态
- 对比度基线：文本与背景不低于 `4.5:1`
- 空态要求：无数据/异常场景必须提供可执行下一步入口

## 5) 权限与审计

- 权限边界：成员仅可操作所属家庭空间内资源
- 跨家庭访问：必须拒绝，且不泄露目标资源敏感信息
- 审计字段（关键写操作）：操作者、对象、动作、时间、结果、追踪编号
- 关键覆盖动作：确认提议、发布记录、邀请成员、上传失败

## 6) 决策编号引用规范（代码注释/评审）

- 引用格式：`D-xx`
- 当前常用编号：
  - `D-02` 设计稿作为实现输入
  - `D-03` 主题能力最小范围
  - `D-04` 权限与审计最小边界
  - `D-06` 量化阈值默认值 + 可配置
  - `D-07` 设计稿路径命名冻结 + 主题跨端同步
- 多决策引用写法：`遵循 D-02, D-03`

## 7) 实施前最小检查清单

- [ ] 设计稿文件在 `design/` 且版本命名符合规则
- [ ] 页面样式已对齐 `design/token.css`，无主题硬编码
- [ ] 主题切换已实现跨端同步读写路径
- [ ] 上传校验按 `10MB + jpg/png/webp` 执行
- [ ] 首屏、反馈时延、对比度基线有可验证证据
- [ ] 权限拒绝与审计记录在关键路径已打通
- [ ] 关键实现注释已标注对应 `D-xx`
