## ADDED Requirements

### Requirement: 生活提议创建规则
系统必须允许家庭成员创建生活提议，且提议标题与计划日期为必填字段，初始状态为待确认。

#### Scenario: 缺失必填字段创建提议
- **WHEN** 成员提交缺失标题或计划日期的提议
- **THEN** 系统必须拒绝创建并返回参数不完整提示

### Requirement: 单次确认即通过
系统必须在任意一位家庭成员首次确认后将提议状态更新为已通过，后续重复确认不应重复触发通过逻辑。

#### Scenario: 提议重复确认
- **WHEN** 提议已为已通过状态且成员再次确认
- **THEN** 系统必须返回“提议已通过”并保持状态不变
