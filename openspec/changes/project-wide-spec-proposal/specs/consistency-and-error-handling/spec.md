## ADDED Requirements

> 约束强度：`MUST`（必须满足），`SHOULD`（推荐满足，若不满足需说明理由），`MAY`（可选增强）。

### Requirement (MUST): 自动草稿幂等约束
系统必须为提议与自动草稿关系建立唯一约束，确保同一提议最多对应一条自动草稿。

#### Scenario: 并发确认同一提议
- **WHEN** 多个成员近同时确认同一提议
- **THEN** 系统必须只保留一条自动草稿记录

### Requirement (SHOULD): 关键异常用户可恢复
系统应当在重复确认、自动草稿失败、上传失败、筛选无结果等关键异常中提供明确提示与恢复路径。

#### Scenario: 自动草稿失败恢复
- **WHEN** 提议通过后草稿未成功生成
- **THEN** 系统应当提示失败原因并提供重试或补建入口
