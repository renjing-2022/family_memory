## ADDED Requirements

> 约束强度：`MUST`（必须满足），`SHOULD`（推荐满足，若不满足需说明理由），`MAY`（可选增强）。

### Requirement (MUST): 图片上传输入约束
系统必须对上传图片执行格式与大小校验，第一版默认支持 `jpg`、`png`、`webp`，单文件大小上限为 10MB；不满足约束时拒绝上传并返回可读错误提示。

#### Scenario: 上传超限文件
- **WHEN** 成员上传超过大小限制或不支持格式的图片
- **THEN** 系统必须拒绝上传并提示允许格式与大小限制（10MB，jpg/png/webp）

### Requirement (SHOULD): 上传失败不阻塞核心记录
系统应当在图片上传失败时允许成员继续保存标题与日期，支持后续补传。

#### Scenario: 上传失败后保存记录
- **WHEN** 图片上传失败但成员继续提交记录表单
- **THEN** 系统应当保存记录最小必填字段并提示可稍后补传

### Requirement (MAY): 媒体资源生命周期治理
系统可以为媒体资源建立记录关联与清理策略，降低无法追踪的孤立资源风险。

#### Scenario: 记录删除后的资源清理
- **WHEN** 关联生活记录被删除或永久失效
- **THEN** 系统可以按清理策略将无引用媒体标记并在约定周期内清理
