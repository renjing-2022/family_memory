## ADDED Requirements

> 约束强度：`MUST`（必须满足），`SHOULD`（推荐满足，若不满足需说明理由），`MAY`（可选增强）。

### Requirement (MUST): 家庭空间操作权限边界
系统必须限制成员仅能在所属家庭空间执行提议确认、记录编辑与成员邀请等操作，跨家庭操作必须被拒绝。

#### Scenario: 跨家庭写操作
- **WHEN** 成员尝试编辑非所属家庭的生活记录
- **THEN** 系统必须拒绝操作并记录权限失败事件

#### Scenario: 未授权访问响应规范
- **WHEN** 成员请求不存在访问权限的家庭资源
- **THEN** 系统必须返回统一无权限响应且不泄露目标资源敏感信息

### Requirement (MUST): 关键写操作审计记录
系统必须对确认提议、发布记录、邀请成员、上传媒体失败等关键操作记录审计字段，包括操作者、对象、动作、时间、结果与追踪编号。

#### Scenario: 成员确认提议
- **WHEN** 成员成功确认提议
- **THEN** 系统必须写入完整审计记录并可用于后续排障检索
