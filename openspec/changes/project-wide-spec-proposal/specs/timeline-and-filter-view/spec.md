## ADDED Requirements

> 约束强度：`MUST`（必须满足），`SHOULD`（推荐满足，若不满足需说明理由），`MAY`（可选增强）。

### Requirement (MUST): 时间线展示能力
系统必须在首页按时间倒序展示生活记录，并对草稿状态进行明显标识。

#### Scenario: 时间线包含草稿与已完成记录
- **WHEN** 成员访问家庭时间线首页
- **THEN** 系统必须按时间倒序展示记录且可区分草稿与已完成状态

### Requirement (SHOULD): 多维筛选与空状态
系统应当支持按成员、主题标签和时间范围筛选，并在无结果时展示可行动空状态。

#### Scenario: 筛选无结果
- **WHEN** 成员设置筛选条件后查询不到记录
- **THEN** 系统应当显示空状态并提供发起提议或记录回忆入口
