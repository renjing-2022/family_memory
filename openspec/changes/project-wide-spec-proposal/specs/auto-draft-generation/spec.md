## ADDED Requirements

### Requirement: 提议通过后自动生成草稿
系统必须在提议状态从待确认变为已通过时自动生成一条生活记录草稿，并继承提议标题、计划日期与提议编号。

#### Scenario: 提议首次通过触发自动草稿
- **WHEN** 提议第一次被成员确认通过
- **THEN** 系统必须创建且仅创建一条与该提议关联的生活记录草稿

### Requirement: 提议通过与草稿生成事务一致性
系统必须保证提议通过与草稿创建在同一事务内完成，避免出现已通过但无草稿的不可恢复状态。

#### Scenario: 草稿创建失败
- **WHEN** 提议状态更新过程中草稿创建失败
- **THEN** 系统必须回滚提议状态变更或提供明确可恢复补偿入口
