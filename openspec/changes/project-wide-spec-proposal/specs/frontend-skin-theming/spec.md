## ADDED Requirements

> 约束强度：`MUST`（必须满足），`SHOULD`（推荐满足，若不满足需说明理由），`MAY`（可选增强）。

### Requirement (MUST): 第一版主题模式范围
系统必须在第一版支持浅色主题、深色主题与系统跟随三种主题模式。

#### Scenario: 系统跟随模式生效
- **WHEN** 用户将主题模式设置为系统跟随且操作系统主题变化
- **THEN** 系统必须自动切换页面主题并保持可读性

### Requirement (SHOULD): 主题切换与持久化
系统应当支持用户手动切换主题模式，并将主题偏好同步到账号维度，确保用户在不同终端登录后保持一致主题选择。

#### Scenario: 用户手动切换主题
- **WHEN** 用户从浅色切换到深色并刷新页面
- **THEN** 系统应当仍以深色模式展示

#### Scenario: 跨端主题一致
- **WHEN** 用户在终端甲切换主题后使用同一账号登录终端乙
- **THEN** 系统应当在终端乙应用与终端甲一致的主题模式

### Requirement (MAY): 主题令牌规范遵循
系统可以通过统一主题令牌文件（例如 `token.css`）驱动颜色与语义样式，并逐步减少业务组件中的主题色值硬编码。

#### Scenario: 新增页面使用主题样式
- **WHEN** 前端新增页面或组件并引入主题样式
- **THEN** 样式可以优先引用主题令牌变量并通过令牌一致性检查
